# Contributing

Thanks for considering a contribution.

## Workflow

1. Fork or branch from the default branch.
2. Keep changes focused and easy to review.
3. Add or update tests/docs when behavior changes.
4. Run the relevant local checks before opening a pull request.
5. Open a pull request with a clear summary and verification notes.

## Pull Request Checklist

- [ ] The change has a clear purpose.
- [ ] Documentation or README examples were updated if needed.
- [ ] Tests, smoke checks, or manual verification were run where practical.
- [ ] No secrets, tokens, private paths, or generated build artifacts are committed.
