# Security Policy

Thanks for helping keep this project safe.

## Reporting a Vulnerability

- Please do **not** publish sensitive exploit details in a public issue.
- If GitHub private vulnerability reporting is available for this repository, use it first.
- Otherwise, open a minimal public issue that says a security report is available, without secrets, tokens, private data, or exploit payloads.

## Scope

This is a small public project maintained on a best-effort basis. Security fixes are prioritized for the default branch and the latest published release, when a release exists.
